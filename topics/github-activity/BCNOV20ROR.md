# Actividad en Github
Al trabajar en equipo es importante que todos los interantes  participen haciendo Commits (CM), Pull Requests (PR) y/o, Code Reviews (CR). El objetivo de este reporte es motivar la participación de todos los BrightCoders. No se trata de ver quién hace más o menos contribuciones, sino como se mencionó, que todos participen. 

Si alguno de los integrantes no tiene muchas participaciones, motívalo o dale oportunidad de que contribuya. El algunas ocasiones algunos participantes no contribuyen al igual que otros pues no sienten mucha confianza, pero con la práctica y ayuda de todos, su confianza puede mejorar!

Al inicio del entrenamiento solamente se trabaja en Commits, pero más adelante comenzarás a hacer Pull Requests (PR) y Code Reviews (CR)

Ek reporte muestra el total de contribuciones por semana así como el total durante todo el entrenamiento y se actualiza por lo menos 1 vez por semana.
